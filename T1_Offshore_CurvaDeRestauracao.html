Metodologia¶

No desenvolvimento deste trabalho iremos criar um método de solução para o problema de uma unidade flutuante que oscila conforme a variação a maré, considerando apenas o problema bidimensional, com a oscilação na direção near e far.

É criada a rotina da catenária para variação do ângulo theta e da projeção horizontal total. Em cada iteração obtém-se um novo valor da força horizontal pelo algoritmo de Newton-Raphson.

Calcula-se a curva de restauração (Deslocamento Horizontal x Força de Restauração) para um riser isolado e para o sistema composto por dois risers para os pontos de um vetor delta x, que varia 10% da lâmina d'água em cada direção, em incrementos de 10 m.

O Problema¶

Cada linha pesa 0,2165kN/m na água e possui um comprimento total de 2800m, sendo instaladas com tração horizontal de topo igual 400kN na posição de projeto.

A profundidade da água é igual a 500m e considera-se que as conexões das linhas com a unidade flutuante estão localizadas na superfície da água.

Pede-se:

a) Determinar a curva de restauração de uma linha de ancoragem isolada.

b) Determinar a curva de restauração do sistema de ancoragem composto pelas duas linhas.

Obs.: Considerar um offset máximo da UF igual a 10% da lâmina d’água para os sentidos “near” e “far”. Apresentar as curvas na forma de tabela e gráfico considerando incrementos de 10m até o offset máximo.

Utiliza-se a seguinte formulação para os parâmetros do problema:

$$ a = \frac{F_x}{q}$$$$ x_s = a \cdot cosh^{-1} \biggl(\frac{y + a}{a}\biggr) $$$$ S = a \cdot senh\biggl(\frac{x_s}{a}\biggr) $$$$ W = q \cdot S $$$$ T = \sqrt{F_x^2 + W^2} $$$$ \theta = tg^{-1}\biggl(\frac{F_h}{W}\biggr) $$$$ x_h = S_t - S $$$$ x_t = x_h + x_s $$

Catenária¶

Para começar o desenvolvimento do código primeiramente precisamos definir a função do Algorítimo de Newton-Raphson e para isso precisamos de uma função de derivação sendo assim, foi-se escolhido a Derivada Central pois é a que tem menos erros das opções de derivação numérica.

Derivada Central¶

Definiremos agora a função de Derivação Central a ser utilizada pelo Algorítimo de Newton-Raphson:

def df(f, x, h):
    """Derivada Central:
    Calcula uma aproximação da primeira derivada de f, nos pontos x, usando passos de tamanho h.
    :param f: Função
    :type f: func
    :param x: Pontos
    :type x: float, int or array
    :param h: Passo
    :type h: float
    :return: Derivada de f no ponto x
    :rtype: float or array
    """
    return (f(x+h) - f(x-h))/(2*h)

Algorítimo de Newton-Raphson¶

Definiremos agora a função que realizará o Algorítimo de Newton-Raphson para encontrar as raízes de uma função:

def newton(f, x0, Y=0, prec_x=1e-6, prec_y=1e-6, maxsteps=100):
    """Algorítimo de Newton-Raphson:
    Calcula o ponto x0 tal que f(x0)=Y
    :param f: Função
    :type f: func
    :param x0: Ponto de Partida
    :type x0: float, int
    :param Y: Valor desejado da função, defaults to 0 (Raíz)
    :type Y: float, int
    :param prec_x, prec_y: Precisão nos valores de x e y, defaults to 1e-6
    :type prec_x, prec_y: float
    :param maxsteps: Número Máximo de Iteraçãoes, defaults to 100
    :type maxsteps: int
    :return: Lista dos valores de x calculados
    :rtype: list
    """
    x_list = []
    for i in range(maxsteps):
        passo = (f(x0)-Y)/df(f, x0, prec_x)
        x = x0 - passo
        x_list.append(x)
        if abs(passo) < prec_x or abs(f(x)-Y) < prec_y:
            return x_list
        x0 = x
    raise "O Número Máximo de Iterações foi atingido"

Definiremos agora a função que realizará as rotinas necessárias para resolver o problema da catenária.

É variado o ângulo Theta e o comprimento total da projeção horizontal. Dentro desse loop é calculada a força horizontal com Newton-Raphson.

Para chutes iniciais de Theta e Xt foram adotados valores pequenos que permitem a convergência. O algoritmo encontra indeterminação se esse valor for determinado como '0' (zero).

In [5]:
def catenaria(St, Xt, y, q, theta, prec=1e-3):
    """Catenária com rotina de Newton-Raphson:
    Soluciona o problema de Riser apoiado por cabo
    :param St: Comprimento Total do Riser (m)
    :type St: float, int
    :param xt, y: Distância entre os apoios (m)
    :type xt, y: float, int
    :param q: Peso por unidade de comprimento (kN/m)
    :type q: float, int
    :param theta: Chute inicial do ângulo entre a resultante e a vertical (°)
    :type theta: float, int
    :param prec: Precisão do Resultado, defaults to 1e-3
    :type prec: float
    :return: Força Horizontal no apoio(kN); Ângulo entre a resultante e a vertical(rad); Força Total no apoio(kN)
    :rtype: float
    """
    # chutes iniciais de theta e Fx
    theta_i = 1
    theta_f = theta
    Fx_i = 1
    # definindo Xt_i somente para passar no primeiro while
    Xt_i = 1
    # primeiro loop, eStimando y e Xt2 a partir de um valor arbitrado de theta
    while abs(theta_f-theta_i) > prec:
        theta_i = theta_f
        y_i = y
        Xt_2 = Xt
        # segundo loop, estimando xs, s, W, theta, xh e Xt1 a partir de um valor arbitrado de Fx
        while abs(Xt_i-Xt_2) > prec:
            a = Fx_i/q
            xs = a*np.arccosh((y_i+a)/a)
            s = a*np.sinh(xs/a)
            W = q*s
            T = np.sqrt(Fx_i**2 + W**2)
            xh = St-s
            # trocando theta para próxima iteração
            theta_f = np.arctan(Fx_i/W)
            # trocando Xt para próxima iteração
            Xt_i = xh+xs
            # definindo a função G(Fx)=Xt
            def G(Fx): return (xh + ((Fx/q)*np.arccosh((y_i*q+Fx)/Fx)))
            # refazendo o processo utilizando newton-raphson para obter o novo valor de Fx
            lista = newton(G, x0=Fx_i, Y=Xt_2)
            Fx_i = lista[-1]
    return Fx_i, theta_f, T

Conclusões¶

Quanto à curva de restauração para a linha isolada, é possível observar que a curva se comporta praticamente exponencialmente conforme o a unidade flutuante se desloca. Quanto mais distante da posição de equilíbrio, maior a força restauradora, uma vez que maior a tração na linha, e a componente horizontal aumenta buscando voltar à posição inicial. Analisando uma linha isolada, se a unidade flutuante se deslocar para outro sentido, essa linha irá "relaxar", diminuindo a componente horizontal da força que busca o equilíbrio.

A geometria, para um offset de 10% da lâmina d'água nos sentidos "near" e "far" se altera. Observa-se a distância horizontal em cada posição sendo:

No equilíbrio:

xh = 1351.699m

xs = 1330.330m

Xt = 2682.029m

Para dx = +50m

xh = 1401.699m

xs = 1330.330m

Xt = 2732.029m

Para dx = -50m

xh = 1301.699m

xs = 1330.330m

Xt = 2632.029m

Sendo xh a distância do riser em solo, xs a projeção horizontal do riser suspenso e Xt o comprimento horizontal total do riser ao TDP.

Já considerando o sistema de ancoragem, considerando a força de restauração para voltar ao equilíbrio como a diferença das forças nas duas direções, nas duas linhas, temos que o valor da força restauradora diminui, uma vez que enquanto uma linha trabalha para retornar ao equilíbrio, a outra possui componente horizontal no sentido contrário. Então, a força que se tem é a diferença entre os valores +dx e -dx.

O estudo das forças restauradoras se mostram de grande importância para que na unidade flutuante, ao atingir uma determinada distância, saiba qual força deve ser aplicada na linha para manter a plataforma em equilíbrio
